# 关于我是怎样在LLM wiki的辅导下为讲座做准备的

<div class="post-info">
  <time>2026-07-07</time>
  <span class="post-tag">pkm</span>
</div>

最近沉迷LLM wiki，几乎已经把这玩意儿当成了我的剪藏库。一开始只是满足了我的知识囤积癖，最近的一次实践让我忽然发现了它带给我的意义之一。

## 从剪藏到伴读

一开始是因为想去听一位教授的讲座，于是将教授接受的访谈剪藏了进去，进而去读了一篇重要著作的序章。我把序章剪藏进了LLM wiki，同时在等待它漫长处理的过程中，亲自读了一遍这篇文章——令人震惊的是，LLM wiki的读取速度比我肉眼读还慢，但仿佛是一个伴读的伙伴，让我能够在读完后对照它的总结，了解我的关注点和一般性的总结有何差异——**了解自己在意的方向并对整体有个更全面的认识**，是在它伴读的过程中我获得的最大收获。

## 概念提取与文献追溯

其实伴读这件事在 NotebookLM 出现时，我就已经尝试过，但LLM wiki给我带来最受用的地方是，它会把文章中提出的重点概念、重要实体、参考文献提取出来，这也是我自己在精读文章时很喜欢做的。一方面，感兴趣的概念可以之后进一步去深挖；另一方面，也很适合去追溯作者和支撑这篇文章的其他文献。

## 让LLM wiki帮我准备讲座提问

接着，我进行了第一次LLM wiki应用的尝试（此前我只是把它当成一个大型剪藏库）——把自己近期的研究方向告诉了LLM wiki，让它**帮我考虑一下讲座时可以向教授问什么样的问题**。说实话，讲座时适合问的问题已经困扰了我很多年，我不能接受去问那种自己查查资料就能得到问题的答案，但又担忧问出一些过于难以回答的问题使对方难堪。我发现这实在是太适合LLM wiki了！它直接给了我三五个可以提问的方向，还结合了讲座的信息，告诉我如果有机会怎样进一步询问。

## 收获：思考流程的内化

我照着它给我的问题在心里想象自己去提出这些问题，瞬间觉得远强过自己在教室里临时想到的那些傻傻的问题。其实我觉得预制一个讲座提问并不是终点，对我而言最大的收获是在过程中了解到了一整套思考的流程——如果哪天我能在讲座的过程中完成这一套思绪，那么我一定能想到更加贴近主题，而又富有深度的问题，一定能造就更加深入的讨论！
