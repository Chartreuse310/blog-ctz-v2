# 关于我的最终版（×7.1）知识组织方法——Solution Z的第一版设计

<div class="post-info">
  <time>2026-07-03</time>
  <span class="post-tag">PKM</span>
  <span class="post-tag">Solution Z</span>
</div>

最近在TRAE AI创意大赛的推动下，强行梳理了最近的知识组织方式，忽然发现这其实是一种结合了PARA系统、卡片笔记写作法、子弹笔记、Obsidian作者用法的巨型缝合怪！

这里面也有亿点点🤏我自己的思考，主要差异在于我是一个心理上更希望体系能**长期主义**一些的人，因此在我的心目中”领域（Area）“是比”项目（Project）“重要的。

因此我进化后的系统（我称为”Solution Z“，即”解答Z“——因为没想到特别好的中文念法所以我一般用英文，这样显得比较酷）如下：

1. **Area**：长期耕耘的领域，这是一个主要的方向，让我能更好地看清自己在做的Project是在深耕主线，还是在扩张副线。
2. **Project**：一个有主题的项目，有明确的目标，但不一定能轻易打勾。Project关注的是项目整体的进展，决定这件事是否有意义，且在持续推进，但并不会直接进入细节。
3. **Task**：广泛的想做的事，它可以直接从属于Area，但更多情况下来源于Project，当它不属于任何时，这是一件不重要且紧迫或不重要且不紧迫的事项，我们需要考虑外包或放弃。
4. **Info**：客观的信息。区别于Note，这是满足了赛博信息收集癖的粮仓，这里像是个人的小型wiki，可以引入资料、整理观点、形成体系。
5. **Note**：来源于**我**的思考。这是产出的核心，它可以关联Area、Project、Task、Info和任何其他Note。
6. **Record**：记录，以时间为单位对Area、Project、Task、Info、Note进行索引，以便快速回顾。

## Solution Z 与 PARA系统 的对应关系

| Solution Z | PARA     | 差异                                                                                                    |
| ---------- | -------- | ----------------------------------------------------------------------------------------------------- |
| Area       | Area     | Solution Z 的 Area 比 Project 更重要，它决定了长期主义的发展方向。                                                    |
| Project    | Project  |                                                                                                       |
| Task       |          | 通过直接能做的事情让Project能真正得以执行。                                                                             |
| Info       | Resource |                                                                                                       |
| Note       |          | 我的思考，产出的核心。Solution Z 里没有直接将这些 Note 分类到 Area 或 Project 里，因为它们可以复用在不同 Project 中，很多时候还跨越了 Area。 |
| Record     | Archive  | Solution Z 里的 Record 更像是一种索引，而非Archive那样类似回收站的作用。                                                 |

我首先选择在Obsidian里尝试使用这个系统，目前来看解决了我几个生活中的问题：

1. 追踪**领域**的进度。
2. **项目**进度留档。
3. **任务**管理——通过在任务里写待办明确下一步怎么做，而不至于直接卡住宕机（这是我作为一个拖延症的重要借口）。
4. **信息**管理——作为一个信息饕餮，我除了拿Obsidian整理书影音，甚至还搞出来过通讯录、番剧列表、live歌单等等，这是我生活中增加幸福感不可不品的一部分。
5. **笔记**管理——说这是整个系统的核心、我的外置大脑也不为过，多年来在flomo和各种Obsidian闪念插件之间反复横跳的万恶之源就是我想建立一种类似卡片笔记写作法的思考库，但**一旦内容过多后就什么都找不着了**的痛我想每一个长期实践过这种方法的人都懂。因此我通过frontmatter使每一个想法都能溯源（是因为领域学习产生的？项目中产生的？任务中产生的？看书/看电影/听音乐产生的？并且标注tags以便聚类我关心的主题，也方便在Obsidian中逆向统计）。
6. **记录**——这部分的思路来源于子弹笔记，但距离我阅读《子弹笔记》已经过去了很久，因此现在的版本是我的魔改版。这部分目前只跑通了行动和日记（行动是从时间轴日记转化而来的，通过一个文件一个行动的方法记录每日行程，也方便可视化），到底要怎么又好看又方便地看周复盘、月复盘、年复盘？或者我是否真的需要它们呢？——这点我现在还没有想清楚。
